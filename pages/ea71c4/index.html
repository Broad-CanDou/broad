# Shell变量

# 1. 定义变量因遵循以下规则

  1. 变量名和等号之间不能有空格
  2. 命名只能使用英文字母，数字和下划线，首个字符不能以数字，标点符号开头
  3. 中间不能有空格，可以使用下划线
  4. 不能使用标点符号
  5. 不能使用bash里的关键字

# 2. 变量的类型

  1. 🚀局部变量🚀 在脚本或命令中定义，仅在当前shell实例中有效，其他shell启动的程序不能访问局部变量。
  2. 🚀环境变量🚀 所有的程序，包括shell启动的程序，都能访问环境变量，有些程序需要环境变量来保证其正常运行。必要的时候shell脚本也可以定义环境变量。
  3. 🚀shell变量（特殊变量）🚀 shell变量是由shell程序设置的特殊变量。shell变量中有一部分是环境变量，有一部分是局部变量，这些变量保证了shell的正常运行
  4. 🚀全局变量🚀 就是指变量在当前的整个 Shell 进程中都有效。每个 Shell 进程都有自己的作用域，彼此之间互不影响。在 Shell 中定义的变量，默认就是全局变量

# 举例

  1. 局部变量: 知道其他语言的小伙伴们都知道局部变量 那只能在定义方法的内部/定义函数的内部进行使用，那如果我们在方法/函数的外部使用是无法使用的。 但是对于Shell 就很奇怪 我们在函数中定义变量在外部也可以使用的：如下
    #!/bin/bash
   function test(){
    a=99;
   }
   test
   echo $a

[root@VM-16-17-centos test]# bash tests.sh

所以证明即使 a 变量在函数test中，但是他的作用域就是全局，那怎么样才能变为局部呢？？ 🚀 方法: 想要成为局部变量 只需要在加上 local 关键字。比如：local a=99;

  1. 全局变量: 全局变量的作用域是当前shell进程(会话),而不是当前shell脚本文件使用source命令（也称为点命令) 例如 source tests.sh这样在一个shell进程(会话)中全局变量在多个文件中是有效的。
  1. 环境变量: 上述讲到全局变量是在当前的shell进程中起作用，但是对于其他shell进程/子进程是无效的。那么这样我们就需要环境变量。环境变量创建时所处的shell进程是父进程，在父进程中创建一个进程称这个心进程为子进程。那么对于环境变量 可以通过父进程传给子进程，同样传给孙进程也阔以。 🚀那是如何创建子进程？如何创建环境变量呢？不要着急 往下看！

  1. 特殊变量:
    1. $n: 传给脚本或者函数的参数 n代表一个数字列如:
    $1,$2,$3.....等最多到10。$0的是当前文件名
    1
    1. $*: 传递给脚本或函数的所有参数。
    2. $@: 传递给脚本或函数的所有参数。
    3. $?: 上个命令的退出状态，或函数的返回值。上个命令执行成功返回 0 不成功返回随机的一个值。 注意: 第二个和第三个的区别在于不被双引号包含时，都以1,2,3的形式输出所有参数，被双引号包含时，"*" 会将所有的参数作为一个整体；"@" 会将各个参数分开，以换行形式输出所有参数。

# 3. 使用变量名

  变量名的使用有两种简单的方式:
  1. echo $test    #加上$符号后面跟变量名
  2. echo ${test}  #加上${}里面跟变量名
1
2
3

🚀 说明: 第二种方式，其实加不加都行。但是呢在有些地方需要加上使解释器识别变量。

# 举例

#!/bin/bash

for param in A B C
do
  echo "输出 ${param}CanShu"
done
