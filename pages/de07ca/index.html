冒泡排序

# 冒泡排序

# 冒泡排序推理

现给定一个数组 int num [] = {27,37,78,66,13}

# 推理图

# 规律总结

  • 趟数(轮数) 由上图可知我们的数组有五个元素，那一共的趟数也就是 5-1=4 一共是四趟。
    那我们元素有n个，那躺数就是 n-1。
  • 交换次数
    第一轮：元素个数5-1=四次
    第二轮：元素个数5-2=三次
    第三轮：元素个数5-3=二次
    第四轮: 元素个数5-4=一次

交换次数=4+3+2+1 = 10 = (n - 1) + (n - 2) + ... + 2 + 1 = n * (n - 1) / 2

# 代码最坏情况

int temp=0;
for (int i=1;i<num.length;i++) {
    for (int j=i;j>0;j--){
        if(num[j-1]>num[j]){
            temp=num[j];
            num[j]=num[j-1];
            num[j-1]=temp;
        }
    }

}

场景: 对于大数据量进行排序的时候，尽量不要使用冒泡排序会大大降低效率。

# 代码最优情况

int temp=0;
for (int i=1;i<num.length;i++) {
    boolean isF=false；
    for (int j=i;j>0;j--){
        if(num[j-1]>num[j]){
            temp=num[j];
            num[j]=num[j-1];
            num[j-1]=temp;
            isF=true;
        }
    }
    if(!isF){
        //那就是代表元素已经是排好的并没有进行交换
        return;
    }

}
